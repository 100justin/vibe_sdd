# 연구 결과: 오늘의 할 일 웹 앱

## 결정된 사항

- FastAPI를 사용해 REST API와 HTML 페이지를 함께 제공한다.
- Jinja2 템플릿과 바닐라 JavaScript를 조합해 단일 페이지 UI를 구현한다.
- 데이터베이스는 SQLite를 사용하며, 경로는 .env의 DATABASE_URL로 관리한다.
- 테스트는 pytest와 FastAPI TestClient를 사용한다.

## 기술 선택 이유

### FastAPI
- Python 기반 웹 프레임워크로 구현 속도가 빠르고 API 문서화가 편리하다.
- Pydantic 기반 검증으로 빈 제목 같은 입력 오류를 쉽게 처리할 수 있다.

### SQLite
- 프로젝트 범위가 소규모이고 별도 데이터베이스 서버가 필요 없기 때문에 적합하다.
- 파일 기반 저장소라 초기 구현과 테스트가 간단하다.

### Jinja2 + 바닐라 JavaScript
- 별도 빌드 도구 없이도 빠르게 화면을 구성할 수 있다.
- fetch를 통해 API와 통신하고 페이지 새로고침 없이 UI를 갱신할 수 있다.

## 구현 주의사항

- 빈 제목은 서버에서 422 응답으로 처리한다.
- 완료 토글은 PATCH 요청으로 처리하고, 프론트엔드는 즉시 목록을 다시 그린다.
- 삭제는 확인 절차를 UI에서 구현하고, 삭제 후 목록을 다시 렌더링한다.
- 남은 할 일 개수는 클라이언트에서 계산하거나 서버에서 제공하는 방식 중 하나로 통일한다.
